Declaring variables

In JavaScript, there are data types with unchangeable values. However, when working with large-scale programs, storing data and intermediate calculation results is often necessary. To accomplish this, variables are used.

What is a variable?

A variable is a special place in memory where you can store data to access it later. It's like a box with a gift inside!

Every variable has a name to distinguish it from other variables. It is possible to access a value by its variable name.

Variables are one of the most often utilized elements of programs, so it is vital to understand and use them well.

How to declare a variable?

Before you start working with a variable, you need to declare (in other words, create) it. JavaScript uses two keywords to create variables:

  • let defines a mutable variable, the value of which can be changed.

  • const declares a constant variable whose value you want to forbid overwriting.

To declare a variable, you must name it using a suitable name. It is good practice to assign a variable a name that describes its contents. Always try to choose meaningful and readable names for variables to make your code easy to understand.

Mutable variables

Let's try to create a mutable variable and understand why it is called that. Let's call it month and put data in it using the = symbol:

let month = "November"; // declare the variable and assign it a value

In JavaScript, variables can store any type of data. In this case, you've saved the string November. You can refer to this string using the variable name:

let month = "November";
console.log(month); // outputs the content of the variable to the console

This code outputs the contents of the variable to the console. In our case, the result will be the string November â€” check it yourself.

The case of the variable name is important: month is not the same as Month.

Now, let's try to change the variable and output its new value to the console:

let month = "November";
month = "December";

console.log(month); // December

As you can see, the value of the variable has changed. Now, it stores the string December.

Constants

Using const, we can declare immutable variables. For example, we can declare the immutable variable language and add the string JavaScript as its value:

const language = "JavaScript";

If we wanted to change the value of a variable created with the const keyword, we would get an error:

const language = "JavaScript";
language = "PHP"; // error

Now that you know about variables, you can create more complex and interesting programs.

Other ways to declare variables

In older scripts, you may notice that the keyword var is used instead of let. This keyword served to declare variables before let and const were introduced in the ECMAScript 6 (ES6) standard. Its syntax looks like this:

var age = 21;

While variables declared with let or const are limited to the block in which they are defined, variables declared as var are accessible throughout the entire function in which they are declared, even outside the block in which they are declared. This can lead to unintended variable pollution. Additionally, variables declared with var can lead to variable hoisting and potential bugs or errors due to accidental redeclaration. For these reasons, it's generally recommended to use let or const instead of var in modern JavaScript code, as they offer more predictable behavior.

You can also find code where variables are declared without keywords:

age = 21;

This is not good practice, either. In the future, we will consider why it is a bad idea to declare variables without keywords or with var, but for now, focus on using let and const.

Conclusion

Variables are a fundamental concept in programming, allowing developers to store and manipulate data in an organized way. With variables, it is possible to create more complex and exciting programs. Remember to choose meaningful and readable names for your variables to make your code easy to understand. Lastly, it is crucial to use the correct syntax to declare variables as mutable (let) or immutable (const); each of these types has its peculiarities. You will be one step closer to becoming a proficient programmer by mastering variables.
